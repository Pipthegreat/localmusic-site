DOM Pérignon - Privacy

Last updated: 2026-06-12

DOM Pérignon stores nothing on a server, sends nothing to anyone, and tracks nothing about you. The whole policy fits on this page.

What we store

A handful of preferences, all in chrome.storage.local on your machine:

  • The overlay you've selected, if any (e.g. champagne, ants).
  • The theme world you've selected, if any (e.g. tubular, arcade).
  • Whether the master Active toggle is on or off.
  • Whether the green >_ override is on.
  • Whether the red >_ override is on.

Versions before 1.5.3 used a single combined selection key (activeHack). On first launch after upgrading, that legacy value is silently migrated into whichever of the two new slots it belongs to and the old key is cleared.

This data never leaves your browser. We have no servers, no analytics, no third-party tracking, no account.

What we read from the pages you visit

DOM Pérignon needs host_permissions: <all_urls> so it can render whichever effect is active on whichever page you're viewing. Different effects look at slightly different things in the page DOM:

  • Logo Bounce House reads <link rel="icon"> and <link rel="apple-touch-icon"> tags, the og:image meta tag, and images inside <header>, <nav>, or elements with "logo" in their class or id. Falls back to the site's /favicon.ico, or to the first letter of the hostname if no logo can be found.
  • Watchers scans for things to ogle - images, video, picture, buttons, headings (h1/h2/h3), paragraphs, blockquotes, and avatar/profile-style elements - and reads their on-screen positions and sizes (via getBoundingClientRect) so the eyes can sit where they should. A MutationObserver watches for matching elements added while the hijink is active.
  • Gravity Well looks at images, buttons, headings (h1/h2/h3), and link-styled buttons so they can fall.
  • You Got This, Champagne Bubbles, Ants, DVD Bouncer, and Aquarium don't read page content - they paint over it.
  • The green >_ and red >_ overrides don't read page content either. The green override paints a falling-glyph overlay on every site; on claude.ai it loads a more elaborate stylesheet targeting Claude's chat UI. The red override does the same with a blood-red palette and its elaborate version applies on chatgpt.com.
  • The theme worlds — Tubular, Arcade, Buzzbin, Vaporwave, Press, Blueprint, Chalkboard — are CSS only. They don't read page content; they apply a stylesheet scoped under html.dp-<name>-on that recolors and re-types whatever the page already shows.

Nothing read from any page is transmitted anywhere. Information stays in the browser, just long enough to render the visual effect.

External network calls

When you open the DOM Pérignon popup, it loads the Fraunces, Inter, and JetBrains Mono fonts from Google Fonts (fonts.googleapis.com) so its own UI displays in the intended typefaces. Several hijinks load additional typefaces from the same source on activation:

  • You Got This — Fraunces, for the drifting quotes.
  • Tubular — Fraunces.
  • Arcade — Press Start 2P.
  • Buzzbin — Anton and other display sans for the sticker headlines.
  • Vaporwave — DM Serif Display.
  • Press — a newsprint serif.
  • Blueprint — Architects Daughter.
  • Chalkboard — Caveat.

Google may log these requests per its own privacy policy. No information about you or the page you're on is sent — only standard requests for the font files. No other DOM Pérignon code makes any external network call; the overlays and >_ overrides load their CSS from inside the extension bundle, not the network.

Permission justifications

  • storage - to remember the five preferences listed above between browser sessions.
  • host_permissions: <all_urls> - because the user explicitly picks what to apply (one overlay, one theme world, the green >_, or the red >_) and that effect renders on whichever page the user is viewing.

Changes to this policy

If this policy ever changes (it almost certainly won't, since the extension does so little), the "Last updated" date at the top of this page will change. The current version is always at customprojects.info/dom-perignon/privacy/.
