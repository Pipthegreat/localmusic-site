localmusic / playertests

Five player builds. Pick one.

The player on sfoak got noisy. Same idea each iteration, more code, less reliable. These five pages are clean rebuilds — five distinct architectures rather than five styles. Each runs against the same 15-song corpus so the variable is the player, not the data.

Test corpus: 5 artists × 3 tracks · Alabama Shakes The xx James Blake Khalid Iron & Wine
V1

Embed-Only

Ember on Slate
Spotify iframe-API

A single Spotify iframe at the bottom, controlled via loadUri() + retry. Closest in spirit to the current sfoak player — but with all the accumulated cruft stripped.

Architecture: one controller, one iframe, retry backoff
V2

Dual Stack

Plum on Slate
Spotify SoundCloud

Each row has SP / SC buttons. Pick a source per tap; the other mount hides. Two API connections live in the page; one is always paused.

Architecture: two iframes, visitor picks source
V3

Auto-Failover

Peacock on Slate
Spotify (primary) SoundCloud (fallback)

Spotify plays. If no playback_update arrives in 3s — Connect conflict, blocked iframe, ad-block — the SoundCloud widget quietly takes over.

Architecture: Spotify primary, stall-timer → SC swap
V4

Native Audio

Ash on Slate
HTML5 audio iTunes previews

No iframe. The page fetches 30s preview MP3s from the iTunes Search API and plays them in <audio>. Hand-off links cover full tracks.

Architecture: page owns audio, services are escape hatches
V5

Stacked Cards

Slate-only
Spotify × 15 embeds

Every track is its own embed. Fifteen iframes, lazy-loaded. No controller, no loadUri, no coordination. The page stops trying to be a player.

Architecture: one embed per track, no cross-iframe state
