Player test · V3

Auto-Failover

Spotify is primary. The page arms a stall timer on every play tap — if no playback_update arrives within 3s (Spotify Connect conflict, ad-blocker, blocked iframe), the SoundCloud widget loads the artist's profile and starts the top track. Visitor sees one player; the source quietly swaps.

Thesis: Cross-source resilience is a UX, not a settings toggle. Pick a primary, and degrade silently. Trade: The SoundCloud fallback isn't track-perfect — it plays the artist's top SC track, which may not match what was tapped. Honest about the swap via a small notice.
